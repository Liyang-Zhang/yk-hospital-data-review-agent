# 部署说明

## 1. 部署目标

本项目按前后端双容器交付：

- `yk-review-agent-backend:session6`：FastAPI 后端，容器内端口 `8000`
- `yk-review-agent-frontend:session6`：React/Vite 静态前端，Nginx 托管，容器内端口 `80`
- `snapshot.db`：运行数据文件，默认已准备好，通过只读 volume 挂载给后端

本项目不接管外层 Nginx、域名、HTTPS、鉴权网关和正式发布。IT 可基于端口信息做统一代理或系统嵌入。

## 2. 本地准备

确认已有可运行的 `snapshot.db`：

```bash
python -m yk_review_agent.tools.inspect_snapshot_db --product pgta
python -m yk_review_agent.tools.inspect_snapshot_db --product pgtsr
```

建库属于独立数据准备流程，不属于默认部署步骤。

## 3. 构建镜像

后端：

```bash
docker build -t yk-review-agent-backend:session6 .
```

前端：

```bash
docker build --build-arg VITE_API_BASE=/api/v1 -t yk-review-agent-frontend:session6 frontend
```

如需指定国内镜像源：

```bash
docker build \
  --build-arg PIP_INDEX_URL=https://pypi.tuna.tsinghua.edu.cn/simple \
  --build-arg PIP_TRUSTED_HOST=pypi.tuna.tsinghua.edu.cn \
  -t yk-review-agent-backend:session6 .

docker build \
  --build-arg NPM_REGISTRY=https://registry.npmmirror.com \
  --build-arg VITE_API_BASE=/api/v1 \
  -t yk-review-agent-frontend:session6 frontend
```

## 4. 本地 Compose 验证

准备运行目录：

```bash
mkdir -p deploy/data
cp snapshot.db deploy/data/snapshot.db
cp deploy/backend.server.env.example deploy/backend.server.env
```

启动：

```bash
docker compose -f deploy/docker-compose.server.yml up -d
```

检查：

```bash
docker compose -f deploy/docker-compose.server.yml ps
curl http://localhost:18765/api/v1/health
curl http://localhost:18088/healthz
```

浏览器访问：

```text
http://localhost:18088
```

停止：

```bash
docker compose -f deploy/docker-compose.server.yml down
```

也可以运行本地 smoke：

```bash
scripts/smoke_docker.sh
```

## 5. 离线交付

本地导出镜像：

```bash
docker save yk-review-agent-backend:session6 -o yk-review-agent-backend-session6.tar
docker save yk-review-agent-frontend:session6 -o yk-review-agent-frontend-session6.tar
```

交付文件：

- `yk-review-agent-backend-session6.tar`
- `yk-review-agent-frontend-session6.tar`
- `deploy/docker-compose.server.yml`
- `deploy/backend.server.env.example`
- 本说明文档
- 已验收的 `snapshot.db`

## 6. 服务器部署

服务器准备目录：

```bash
mkdir -p /opt/yk-review-agent/data
cd /opt/yk-review-agent
```

放置文件：

```text
/opt/yk-review-agent/data/snapshot.db
/opt/yk-review-agent/backend.server.env
/opt/yk-review-agent/docker-compose.server.yml
```

加载镜像：

```bash
docker load -i yk-review-agent-backend-session6.tar
docker load -i yk-review-agent-frontend-session6.tar
```

编辑环境变量：

```bash
cp backend.server.env.example backend.server.env
```

至少确认：

```env
SNAPSHOT_DB_URL=sqlite+pysqlite:////app/snapshot.db
HOST_ALLOWED_ORIGINS=http://<server-ip>:18088
LLM_API_KEY=
```

启动：

```bash
docker compose -f docker-compose.server.yml up -d
```

查看状态和日志：

```bash
docker compose -f docker-compose.server.yml ps
docker compose -f docker-compose.server.yml logs -f backend
docker compose -f docker-compose.server.yml logs -f frontend
```

## 7. Docker Run 备选

后端：

```bash
docker run -d \
  --name yk-review-agent-backend \
  --restart unless-stopped \
  -p 18765:8000 \
  --env-file /opt/yk-review-agent/backend.server.env \
  -v /opt/yk-review-agent/data/snapshot.db:/app/snapshot.db:ro \
  yk-review-agent-backend:session6
```

前端：

```bash
docker run -d \
  --name yk-review-agent-frontend \
  --restart unless-stopped \
  -p 18088:80 \
  yk-review-agent-frontend:session6
```

如果使用 `docker run` 启动前端，前端容器内的 `/api/v1` 默认代理到名为 `backend` 的 compose 服务。纯 `docker run` 场景建议由 IT 在外层代理 `/api/v1` 到 `18765`，或改用 compose 主流程。

## 8. 验收清单

基础检查：

```bash
curl http://<server-ip>:18765/api/v1/health
curl http://<server-ip>:18088/healthz
```

业务最小验收：

- `山西妇幼2025年PGT-A周期量和胚胎量分别是多少` 返回 `answer`
- `山西妇幼2025年PGT-A整倍体率是多少` 返回 `answer`
- `2025年PGT-SR送检量是多少` 返回 `answer`
- `按临床指征看一下PGT-SR胚胎整倍体率` 返回受控结果
- `PGT-M 2025年整倍体率是多少` 返回 `refuse`
- `天气怎么样` 返回 `refuse`

交付给 IT：

- 前端端口：`18088`
- 后端端口：`18765`
- API 前缀：`/api/v1`
- 后端健康检查：`GET /api/v1/health`
- 前端健康检查：`GET /healthz`
- 数据挂载：`/opt/yk-review-agent/data/snapshot.db -> /app/snapshot.db:ro`

## 9. 更新和回滚

更新镜像：

```bash
docker compose -f docker-compose.server.yml down
docker load -i yk-review-agent-backend-session6.tar
docker load -i yk-review-agent-frontend-session6.tar
docker compose -f docker-compose.server.yml up -d
```

更新数据：

```bash
docker compose -f docker-compose.server.yml down
cp snapshot.db /opt/yk-review-agent/data/snapshot.db
docker compose -f docker-compose.server.yml up -d
```

回滚时恢复上一版镜像 tar 包和上一版 `snapshot.db`，再重新 `docker compose up -d`。
