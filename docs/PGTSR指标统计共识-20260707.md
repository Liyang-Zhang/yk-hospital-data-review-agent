# PGT-SR 指标统计共识

本文用于说明当前仓库里 `PGT-SR` 已接入的真实统计能力：目前能统计哪些指标、统计共识是什么、底层实际使用了哪些列。

适用范围：

- 当前代码版本的 `PGT-SR` 第一阶段真实执行
- 数据源为 `docs/PGTSR数据统计输出-2025年.xlsx`
- 主要对应实现：
  - `app/yk_review_agent/services/pgtsr_sqlite.py`
  - `app/yk_review_agent/services/query_service.py`
  - `app/yk_review_agent/services/metric_catalog.py`

---

## 1. 当前已支持的 PGT-SR 指标

当前已接入并可从真实执行链路触发的指标共有 6 类：

1. `pgtsr_total_volume`
   - `PGT-SR` 送检量
2. `pgtsr_euploid_rate`
   - `PGT-SR` 整倍体率
3. `pgtsr_quality_overview`
   - `PGT-SR` 检测与质控总览
4. `pgtsr_result_overview`
   - `PGT-SR` 结果分布
5. `pgtsr_cycle_indicator_overview`
   - `PGT-SR` 周期结局总览
6. `pgtsr_next_step_overview`
   - `PGT-SR` 下一步易位筛查总览

当前不应承诺为已稳定支持的能力：

- `PGT-SR` 扩增成功率
- `PGT-SR` 意外发现率
- `PGT-SR` 二阶段不携带率
- 复杂多指标联合统计

## 2. 全局统计共识

### 2.1 数据来源与入库规则

- 主数据 sheet：`2025年-数据`
- 项目补充 sheet：`2025年-项目`
- 导入目标表：`pgtsr_snapshot_raw`

### 2.2 全局过滤规则

所有当前 `PGT-SR` 指标都共享以下基础过滤规则：

- 只统计成功导入的样本行
- 排除 `样本类型` 中包含 `对照` 或 `极体` 的记录
- 只有同时具备以下字段的行才会进入统计：
  - `送检单编号`
  - `样本编号`
  - `送检单位名称`
  - `报告审核时间`

### 2.3 当前支持的筛选维度

- 医院：`hospital_name`
- 时间：优先 `月份`，按日时使用 `报告审核时间`
- 女方年龄：`受检人年龄`
- 男方年龄：`配偶年龄`
- `SR` 临床指征：`临床指征`

### 2.4 当前支持的 breakdown

- `overall`
- `month`
- `quarter`
- `day`
- `qc`
- `result`
- `sr_clinical_type`

但不是每个指标都支持全部 breakdown，需按指标定义理解。

### 2.5 列映射共识

当前 `PGT-SR` 执行使用的是下面这套归一化字段：

| 归一化字段 | 来源列 |
| --- | --- |
| `month_bucket` | `月份`，缺失时回退到 `报告审核时间` 的年月 |
| `created_at` | `报告审核时间` |
| `hospital_name` | `送检单位名称` |
| `cycle_id` | `送检单编号` |
| `sample_id` | `样本编号` |
| `sample_name` | `送检单样本名称` |
| `sample_type` | `样本类型` |
| `patient_age` | `受检人年龄` |
| `spouse_age` | `配偶年龄` |
| `patient_karyotype` | `受检人核型` |
| `spouse_karyotype` | `配偶核型` |
| `qc_conclusion` | `data_QC_conclusion` |
| `cnv_result` | `CNV检测结果` |
| `result_label` | `结果解释` |
| `cnv_hint` | `提示CNV` |
| `result_detail` | `结果说明` |
| `sr_clinical_type` | 优先 `2025年-项目` 的 `临床指征`，否则回退 `2025年-数据` 的 `临床指征` |
| `next_step_screening` | 优先 `2025年-项目` 的 `是否进入下一步易位筛查`，否则回退 `2025年-数据` 同名列 |

说明：

- `sr_clinical_type` 和 `next_step_screening` 当前优先信任项目 sheet。
- 这两个字段已经不是“附加说明列”，而是当前 `PGT-SR` 周期级统计的重要 contract 字段。

## 3. 各指标统计共识

## 3.1 `pgtsr_total_volume`

### 当前统计什么

统计当前筛选范围内的：

- 检测周期数
- 检测胚胎数
- 平均每周期胚胎数

### 当前统计口径

- `检测胚胎数` = 样本记录数
- `检测周期数` = 去重后的 `送检单编号` 数
- `平均每周期胚胎数` = `检测胚胎数 / 检测周期数`

### 支持的 breakdown

- `overall`
- `month`
- `quarter`
- `day`

### 依赖列

- 必需主列：
  - `送检单编号`
  - `样本编号`
  - `送检单位名称`
  - `报告审核时间`
  - `月份`
- 可选筛选列：
  - `受检人年龄`
  - `配偶年龄`
  - `临床指征`

### 当前不承诺什么

- 不等同于“家系数”
- 不区分遗传型构建 / 非遗传型构建
- 不对不同核型做单独统计解释

---

## 3.2 `pgtsr_euploid_rate`

### 当前统计什么

统计当前筛选范围内的：

- 检测胚胎数
- 整倍体胚胎数
- 整倍体率

### 当前统计口径

- 当前 `PGT-SR` 的整倍体率按 `结果解释 == 未见异常` 统计
- `整倍体率` = `整倍体胚胎数 / 检测胚胎数`
- 时间趋势按胚胎样本层面统计，不按周期聚合

### 支持的 breakdown

- `overall`
- `month`
- `quarter`
- `day`
- `age`
- `sr_clinical_type`

### 年龄分层当前共识

- `breakdown=age` 时，当前会同时输出：
  - 女方年龄分层
  - 男方年龄分层
- 当前两套年龄分层都统一沿用三档：
  - `＜35岁`
  - `35-38岁`
  - `＞38岁`
  - `未填写`

### 依赖列

- `样本编号`
- `送检单编号`
- `送检单位名称`
- `报告审核时间`
- `月份`
- `结果解释`
- `受检人年龄`
- `配偶年龄`
- `临床指征`

### 当前共识提醒

- 这里是胚胎层面的整倍体率，不是周期层面的结局。
- 如果用户问的是“周期整倍体结局 / 周期无整倍体率”，应该走 `pgtsr_cycle_indicator_overview`，而不是这个指标。
- 如果用户问“按临床指征看 PGT-SR 胚胎整倍体率”，应保留 `breakdown=sr_clinical_type`，输出 `临床指征 / 检测胚胎数 / 整倍体胚胎数 / 整倍体率`。
- 如果用户只说“按临床指征看 PGT-SR 整倍体率”，但没有说明胚胎层面或周期层面，应先澄清统计对象，不应默认退化成 `总体` 或默认选择周期结局。
- 如果用户只说“35岁以上”但没说明是女方还是男方，当前应先澄清年龄对象。

---

## 3.3 `pgtsr_quality_overview`

### 当前统计什么

统计当前筛选范围内的：

- 检测周期数
- 检测胚胎数
- 平均每周期胚胎数
- 检测成功率
- `NA` 胚胎数
- `PASS / INFO / FAIL` 分布

### 当前统计口径

- `检测成功率` = `CNV检测结果` 非空且不为 `/` 的样本数 / 总样本数
- `NA胚胎数` = 满足以下任一条件的样本数：
  - `data_QC_conclusion == FAIL`
  - `结果解释 == 质控不合格`
- `PASS / INFO / FAIL` 分布按 `data_QC_conclusion` 直接计数

### 支持的 breakdown

- `overall`
- `month`
- `quarter`
- `day`
- `qc`

### 依赖列

- `送检单编号`
- `样本编号`
- `送检单位名称`
- `报告审核时间`
- `月份`
- `data_QC_conclusion`
- `CNV检测结果`
- `结果解释`

### 当前共识提醒

- 这里的“检测成功率”不是“扩增成功率”。
- 当前代码没有为 `PGT-SR` 建立单独的扩增成功率正式口径。
- 如果用户问的是扩增成功率，当前应视为未正式支持，而不是拿检测成功率顶替。

---

## 3.4 `pgtsr_result_overview`

### 当前统计什么

统计当前筛选范围内 `结果解释` 的四类分布：

- `整倍体`
- `嵌合异常`
- `异常`
- `质控不合格`

### 当前统计口径

- 当前完全按 `结果解释` 归一化后的 `result_label` 做分类计数
- 其中对外展示时，会把 `结果解释 == 未见异常` 统一表述为 `整倍体`
- 总体模式下输出四类样本数和占比
- `result` breakdown 下输出四类结果结构分布

### 支持的 breakdown

- `overall`
- `result`

说明：

- `metric_catalog` 里虽然给了 `month / quarter / day` 能力声明，但当前 `query_service` 实现并没有真正按时间拆 `result_overview`。
- 现阶段应把它视为“总体结果分布 + 结果结构 breakdown”能力，而不是完整的时间趋势指标。

### 依赖列

- `样本编号`
- `送检单编号`
- `送检单位名称`
- `报告审核时间`
- `结果解释`

### 当前不承诺什么

- 不等同于 `PGT-A` 的整倍体/异倍体/意外发现口径
- 不基于 `提示CNV` 或 `结果说明` 再细分异常亚型
- 当前对外展示已统一用“整倍体”表述，但底层匹配仍然基于 `结果解释 == 未见异常`

---

## 3.5 `pgtsr_cycle_indicator_overview`

### 当前统计什么

按周期统计“该周期是否存在至少一个整倍体胚胎”。

当前输出的是：

- 检测周期数
- 周期有整倍体胚胎占比
- 周期无整倍体胚胎占比

### 当前统计口径

- 先按 `送检单编号` 聚成周期
- 若某周期下任一胚胎 `结果解释 == 未见异常`，则该周期记为“周期有整倍体胚胎”
- 否则记为“周期无整倍体胚胎”

这意味着它当前回答的是：

- “这个周期里有没有至少一个可归到整倍体口径的胚胎”

而不是：

- 周期整倍体率
- 周期可移植率
- 周期妊娠结局

### 支持的 breakdown

- `overall`
- `month`
- `quarter`
- `sr_clinical_type`

### 依赖列

- `送检单编号`
- `结果解释`
- `月份`
- `报告审核时间`
- `临床指征`

### 当前共识提醒

- 名称里写的是“周期结局”，但当前业务含义更准确地说，是“周期是否存在整倍体胚胎”。
- 如果后续要把它升级成更强的周期结局 contract，需要先单独确认业务语义，不能直接沿用现在的命名做更强承诺。

---

## 3.6 `pgtsr_next_step_overview`

### 当前统计什么

按周期统计“是否进入下一步易位筛查”，以及进入后的构建类型分布。

当前输出状态包括：

- `否`
- `是（评估后符合非遗传型构建）`
- `是（评估后符合遗传型构建）`
- `/`

### 当前统计口径

- 先按 `送检单编号` 聚成周期
- 每个周期取一个 `next_step_screening` 状态
- 当前代码使用 `setdefault`：
  - 对同一周期，默认保留遍历到的第一条状态
  - 不做多条状态冲突校验

### 支持的 breakdown

- `overall`
- `month`
- `quarter`
- `sr_clinical_type`

### 依赖列

- `送检单编号`
- `是否进入下一步易位筛查`
- `月份`
- `报告审核时间`
- `临床指征`

### 当前共识提醒

- 该指标本质是“周期级状态分布”，不是胚胎级比例。
- 当前默认同一周期的 `next_step_screening` 应该一致；如果源数据未来出现同周期多状态冲突，需要单开规则处理。
- `/` 当前被当作一个显式状态保留在结果中，不能静默并入“否”。

---

## 4. 当前最需要继续收口的地方

按优先级，当前最值得继续推进的是：

1. `pgtsr_cycle_indicator_overview`
   - 名称是“周期结局”，但当前实际只统计“是否存在整倍体胚胎”
2. `pgtsr_result_overview`
   - 需要确认是否长期只承诺四类 `结果解释` 分布
3. `pgtsr_next_step_overview`
   - 需要确认同一周期多状态时的正式冲突处理规则
4. `pgtsr_quality_overview`
   - 需要持续强调“检测成功率”与“扩增成功率”不是一回事

---

## 5. 一句话结论

当前 `PGT-SR` 已经不是“只接快照未执行”，而是已经具备一批可真实执行的基础统计能力；但这些能力的核心共识应理解为：

- 先承诺基础规模统计
- 先承诺基于 `未见异常` 的胚胎整倍体率
- 先承诺 `结果解释` 四分类分布
- 先承诺“周期是否存在整倍体胚胎”
- 先承诺“是否进入下一步易位筛查”

不要把它提前表述成完整的 `PGT-SR` 全量业务指标体系。
