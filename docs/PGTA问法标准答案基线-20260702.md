# PGT-A 问法标准答案基线（2026-07-02）

## 目的

本文档用于给当前 `PGT-A` 问法池补一层“标准答案基线”，把**路由正确但答案错误**的问题和正确业务结果对齐。

当前基线只覆盖：

- 当前真实可执行产品：`PGT-A`
- 当前默认联调医院：`山西省妇幼保健院`
- 当前业务审核快照：[PGTA数据统计输出-2025年.xlsx](/home/zhangly/repo/github/yk-hospital-data-review-agent/docs/PGTA数据统计输出-2025年.xlsx)

不覆盖：

- `PGT-SR / PGT-M / PGTAH`
- 跨产品复杂任务
- 尚未和医学部完全对齐口径的指标，如 `扩增成功率`

## 数据源与使用原则

### 优先级 1：`2025年-数据统计`

适用于以下问题的标准答案：

- 按医院、按月、按季度、按年的送检量
- 按医院、按月、按季度、按年的整倍体率
- 检测成功率、NA、PASS/INFO/FAIL
- 结果结构、意外发现

这张 sheet 是业务人工透视后的结果，优先视为当前标准答案来源。

### 优先级 2：`2025年-数据`

适用于以下问题的标准答案：

- 年龄范围筛选，如 `>35岁`、`35-38岁`、`未填写年龄`
- 周期结局类统计
- 暂时无法直接从透视表一眼读出的明细聚合口径

这部分仍然以业务快照为准，但属于“按当前字段定义重新聚合”的结果。

## 当前仍不建议直接作为“标准答案”的题型

以下题型当前可以保留路由回归，但暂不适合写死成标准答案基线：

- `特殊 CNV 提示` 的细分值
  - 当前代码已经能算一轮
  - 但仍建议先和医学部再核对 1Mb~4Mb、4Mb~10Mb、拟常染色体区域等细项口径
- `扩增成功率`
  - 当前字段和阈值口径仍未固定
- `异倍体率`
  - 当前执行按 `异倍体结果（人工处理）` 做窄口径识别
  - 需要再确认是否完全等于业务透视表中的“异倍体结果占比”

## 建议用法

后续开发和回归时，把问题拆成两层：

1. `路由是否正确`
   - 命中的 `metric family` 是否符合预期
2. `答案是否正确`
   - 先对照本文档的标准答案基线
   - 如果和本文档不一致，再回到字段口径和函数计算逻辑排查

这样能把“意图识别问题”和“统计函数问题”真正拆开，不再混在一起调。
