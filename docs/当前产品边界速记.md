# 当前产品边界速记

## 1. 当前数据源

- 当前执行源只使用业务人工审核快照：
  - `PGTA数据统计输出-2025年.xlsx`
  - `PGTAH数据统计输出-2025年-上传微盘.xlsx`
  - `PGTSR数据统计输出-2025年.xlsx`
  - `2025-PGTM类全年输出.xlsx`
- 当前不再使用早期 `statistics_detail.xlsx / statistics_current_table.xlsx` 作为执行源。
- 后续如接正式接口，只替换数据适配层，不重写问答主干。

## 2. 当前真实执行范围

- 当前真实可执行产品：
  - `PGT-A`
  - `PGT-SR` 第一阶段
- 当前仅做快照接入和能力说明的产品：
  - `PGT-AH`
  - `PGTM`

## 3. 当前支持的 PGT-A 统计主题

- 送检量
- 整倍体率
- 质控 / 检测成功率
- 结果分布 / 嵌合与异常结构 / 意外发现率 / 异倍体率
- 周期结局
- 特殊 CNV 提示

## 4. 当前统一筛选维度

- 医院
- 时间范围
- 年龄范围

说明：

- `年 / 月 / 季度` 优先基于业务统计月份字段
- `按日` 基于审核时间字段
- 年龄范围是全局 filter，不是单独主题特例
- 年龄分层是全局 breakdown，不作为独立 metric 路由

## 5. 当前明确不承诺的能力

- `PGT-AH / PGTM` 真实统计执行
- `PGT-SR` 全量业务指标体系
- 全产品汇总执行
- 复杂多指标联合执行
- 扩增成功率正式执行结果
- 临床指征、高龄/流产/种植失败等占比

## 6. 当前路由原则

- 先做问题规范化
- 再做意图识别和追问继承
- 再做可分析性判断
- 最后才进入受控函数执行
- 不能明确落到唯一统计主题的问题，不允许默认兜底到邻近指标
