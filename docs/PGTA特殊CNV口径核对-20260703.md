## 当前代码口径

当前 `pgta_special_cnv_overview` 已按以下前提执行：

- 只统计 `意外发现（人工处理） == 有` 的胚胎
- 统计维度是胚胎数
- 若同一胚胎同时命中多个特殊类型，则各类型分别计一次
- 特殊 CNV 判断只依赖：
  - `提示CNV`
  - `结果说明`
- 不再使用 `结果解释` 作为特殊 CNV 的过滤前提

四个子项当前规则如下：

1. `1Mb~4Mb 综合征相关提示`
   - `提示CNV` 中存在 `~X.XXMb`
   - 且大小满足 `1 <= Mb < 4`
   - 且 `结果说明` 中包含 `包含已知综合征`
   - 且不包含 `无已知综合征`
2. `4Mb~10Mb 且≥50%嵌合且综合征相关`
   - `提示CNV` 中大小满足 `4 <= Mb < 10`
   - 且嵌合比例 `>= 50%`
   - 且 `结果说明` 中满足“包含已知综合征”且不为“无已知综合征”
3. `≥10Mb 且20%~30%嵌合`
   - `提示CNV` 中大小满足 `>= 10Mb`
   - 且嵌合比例在 `20%~30%` 之间
   - 当前不要求综合征支持
4. `拟常染色体区域缺失`
   - `提示CNV` 中包含 `del(X)` 且包含 `p22.33`
   - 或 `结果说明` 中出现 `X染色体p22.33` 且出现 `缺失`
   - 当前只统计缺失，不统计重复

详细规则同步见 [PGTA指标计算说明-20260701.md](/home/zhangly/repo/github/yk-hospital-data-review-agent/docs/PGTA指标计算说明-20260701.md:166)。
