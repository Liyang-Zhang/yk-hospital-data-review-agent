# 多线程开发规划

## 使用方式

本文件用于把后续工作拆成多个短线程，降低单线程上下文膨胀和 token 消耗。

每个新线程建议只聚焦一个主题，并在开头引用：

- [当前产品边界速记.md](/home/zhangly/repo/github/yk-hospital-data-review-agent/docs/当前产品边界速记.md)
- [执行级技术架构方案.md](/home/zhangly/repo/github/yk-hospital-data-review-agent/docs/执行级技术架构方案.md)

这样通常不需要再带整段长背景。

## Session 1：PGT-A 特殊 CNV 口径核对

### 目标

确认 `pgta_special_cnv_overview` 的业务口径是否已经足够稳定，可以进入标准答案基线和 oracle 回归。

### 重点输入

- [PGTA指标计算说明-20260701.md](/home/zhangly/repo/github/yk-hospital-data-review-agent/docs/PGTA指标计算说明-20260701.md)
- [PGTA问法标准答案基线-20260702.md](/home/zhangly/repo/github/yk-hospital-data-review-agent/docs/PGTA问法标准答案基线-20260702.md)

### 下一步

1. 列出 `1Mb~4Mb / 4Mb~10Mb / ≥10Mb / 拟常染色体区域` 当前分别用了哪些列
2. 给出每一项是否已具备唯一业务语义
3. 只对可唯一确定的细项建立 oracle，不强行覆盖全部细口径

### 成功标准

- 至少明确哪些特殊 CNV 项可以进入强回归
- 若不能进入，输出医学部最小确认清单

## Session 2：PGT-A 路由长尾与追问继承（已完成）

### 目标

继续优化单问题口语问法和 3 到 4 跳追问链的稳定性。

### 重点输入

- [V0.2路由语料回归说明.md](/home/zhangly/repo/github/yk-hospital-data-review-agent/docs/V0.2路由语料回归说明.md)

### 完成产物

1. 已完成一轮轻量级路由优化：
   - 补了医院简称、短年份、年龄口语表达等 normalization
   - 补了 `周期量 / 胚胎量`、周期结局、特殊 CNV 等长尾问法
   - 收敛了追问继承，新主题出现时不继续沿用旧 metric
2. 已完成医院数据访问边界：
   - 默认只允许访问当前 `host_context.hospital_id`
   - 只有 IT 显式传入 `accessible_hospital_ids` 或 `can_access_all_hospitals=True` 时，才允许跨医院查询
   - 显式提到未授权医院时直接 `refuse`
3. 已完成轻量回归：
   - 路由相关测试 `39 passed`
   - 覆盖医院权限拒答、跨医院授权放行、长尾问法、追问继承、特殊 CNV smoke

### 当前结论

- 前端默认挂载山西妇幼会话时，`301这边2025年PGT-A周期量和胚胎量分别是多少` 应拒答
- `山西妇幼2025年PGT-A周期量和胚胎量分别是多少` 可正常回答
- `未填写年龄的这些样本，PGT-A结果怎么样` 仍应 `clarify`，不默认映射到具体函数
