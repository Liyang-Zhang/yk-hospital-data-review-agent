# 第一版人工测试说明

## 1. 测试目标

验证当前第一版是否已经具备以下能力：

- 能读取当前业务高质量快照文件
- 能基于真实医院创建会话
- 能回答第一版支持的 `PGT-A` 问题
- 能对模糊但域内的问题进入澄清
- 能对暂不支持的问题明确拒答
- 前端页面可以正常展示文本、表格和图表卡片

## 2. 当前支持范围

当前只支持：

- 数据源：
  - `docs/PGTA数据统计输出-2025年.xlsx`
  - `docs/PGTAH数据统计输出-2025年-上传微盘.xlsx`
  - `docs/PGTSR数据统计输出-2025年.xlsx`
  - `docs/2025-PGTM类全年输出.xlsx`
- 真实执行产品范围：`PGT-A`
- 能力说明范围：`PGT-A / PGT-AH / PGT-SR / PGT-M`
- 时间范围：
  - `年 / 月 / 季度` 优先基于业务统计月份字段
  - `按日` 基于审核时间字段
- 医院范围：当前会话医院
- 年龄范围：支持作为统一筛选条件传入当前 `PGT-A` 单指标执行链
- 问题类型：
  - 送检量
  - 整倍体率
  - 年龄分层整倍体率
  - 质控 / 检测成功率
  - 结果分布 / 嵌合与异常结构 / 意外发现率 / 异倍体率
  - 周期结局
  - 特殊 CNV 提示

当前问答状态分为：

- `answer`：问题支持且可直接执行
- `clarify`：问题属于 PGT-A 数据回顾，但主指标不明确
- `refuse`：问题超出产品范围、数据范围或当前不支持的组合

当前不支持：

- `PGT-AH` 真实统计执行
- `PGT-SR` 真实统计执行
- `PGT-M` 真实统计执行
- 全产品汇总
- 扩增成功率正式执行结果
- 临床指征、高龄/流产/种植失败等占比
- 复杂多指标联合执行

## 3. 启动步骤

### 3.1 启动后端

如需启用 Qwen 模型解析，先创建 `.env`：

```bash
cp .env.example .env
```

并填写：

- `LLM_MODEL=qwen-plus` 或其他 Qwen 系列模型
- `LLM_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1`
- `LLM_API_KEY=你的 DashScope 兼容接口密钥`

未配置 `LLM_API_KEY` 时，会自动回退到本地规则解析。

```bash
micromamba activate yk-review-agent
uvicorn yk_review_agent.main:app --reload --port 18765
```

后端地址默认：

- `http://localhost:18765`

### 3.2 启动前端

```bash
cd frontend
export VITE_API_BASE=http://localhost:18765/api/v1
micromamba run -n yk-review-agent npm run dev
```

前端地址默认：

- `http://localhost:5173`

## 4. 后端接口人工检查

### 4.1 健康检查

访问：

- `http://localhost:18765/api/v1/health`

预期：

- 返回 `status=ok`

### 4.2 metadata 接口

访问：

- `http://localhost:18765/api/v1/demo/metadata`

预期：

- 返回 `product_scope` 包含 `PGT-A（可执行）`
- 返回 `data_source = business_snapshot_bundle`
- 返回 `default_hospital`
- 返回 `snapshot_start` 和 `snapshot_end`

## 5. 前端页面人工检查

打开：

- `http://localhost:5173`

预期：

- 页面可正常加载
- 右上角显示“会话已连接”
- 展示默认医院名称
- 展示时间快照范围
- 左侧推荐问题点击后应直接发送
- 结果区的追问按钮点击后应直接发送
- 图表不是每次都必须出现；当只有单个分组或单个时间点时，允许只显示文字和表格

## 6. 建议测试问题

### 6.1 支持的问题

依次测试以下问题：

1. `看一下当前快照下的 PGT-A 送检量`
2. `2025 年 PGT-A 的整倍体率是多少？`
3. `按年龄分层看一下 PGT-A 的整倍体率`
4. `看一下当前快照下的 PGT-A 质控情况`
5. `看一下 PGT-A 的嵌合与异常结构`
6. `看一下 PGT-A 的周期无整倍体率`
7. `看一下 PGT-A 的特殊 CNV 提示情况`
8. `山西妇幼在25年7月到10月>35岁患者的整倍体率`

预期：

- 每次都返回中文结论
- 返回至少一张结果卡片
- 表格列名与问题相关
- 如果存在两个及以上分组/时间点，图表应能正常渲染
- 如果只有一个分组/时间点，允许不出图
- 回答末尾应带有“当前业务整理的 PGT-A 快照”时间快照说明

### 6.2 暂不支持的问题

依次测试以下问题：

1. `2025 年 PGT-SR 二阶段结果怎么样？`
2. `看一下 2025 年 PGT-M 的病种分布`
3. `看一下 PGT-A 的扩增成功率`
4. `看一下高龄和流产相关临床指征占比`

预期：

- 系统明确提示当前第一版暂不支持
- 不应返回伪造统计结果
- 应给出可继续测试的 PGT-A 推荐问题

### 6.3 需要澄清的问题

依次测试以下问题：

1. `帮我分析一下`
2. `看一下这个医院的情况`

预期：

- 系统不直接返回某个默认指标
- 系统应提示需要先明确主指标
- 应给出可点击的推荐问题

## 7. 当前可对照的结果

使用默认医院 `中国人民解放军医院301医院` 且时间为 `当前快照全部时间` 时，可人工对照这些结果：

- 有效胚胎样本数：`342`
- 周期数：`86`
- 平均每周期胚胎数：`3.98`
- 整倍体数：`156`
- 整倍体率：`45.6%`
- NA 胚胎数：`7`
- 检测成功率：`98.0%`
- 异常样本数：`150`
- 嵌合异常样本数：`29`

如果页面返回结果和这些明显不一致，需要重点检查：

- 是否正确排除了对照/极体样本
- 是否拿错了业务统计月份范围
- 是否使用了错误医院上下文
- 是否遗漏了年龄范围筛选

## 8. 需要重点观察的体验问题

- 点击推荐问题后是否直接发送，而不是只回填输入框
- 发送后页面是否进入加载态
- 结果卡片是否有明显错位或溢出
- 表格在窄屏下是否还能横向滚动
- 拒答时是否足够明确，没有误导用户

## 9. 现阶段已知限制

- 只有一个默认医院上下文，没有前端医院切换器
- 时间解析目前支持 `年 / 季度 / 月 / 日`
- 时间范围支持 `2025年7月到2025年10月` 这类月度区间
- 年龄范围支持 `>35岁 / 35-37岁 / 未填写年龄`
- 没有做用户登录和宿主系统透传
- 没有 PDF 导出
- 复杂多指标请求仍以 `clarify / capability report / refuse` 为主，不做真实联合执行

## 10. 本轮真实链路调试记录

### 10.1 调试问题

- `统计下2025年7月-10月的整倍体率情况`

### 10.2 实际链路结果

- 规范化后问题：
  - `统计下2025年7月到2025年10月的整倍体率情况`
- 识别结果：
  - `product_scope = PGT-A`
  - `time_range = 2025年7月到2025年10月`
  - `breakdown = overall`
  - `focus = rate`
- 最终路由函数：
  - `pgta_euploid_rate`
- answerability：
  - `answer`
- 默认医院 `中国人民解放军医院301医院` 下的实际回答：
  - 检测胚胎数 `860`
  - 整倍体数 `335`
  - 整倍体率 `39.0%`

### 10.3 是否符合预期

这条问题当前判断为“基本符合预期”：

- 符合预期的部分：
  - 时间范围正确识别为 `2025年7月到2025年10月`
  - 主题正确落到 `PGT-A 整倍体率`
  - 最终函数正确命中 `pgta_euploid_rate`
  - 成功返回真实统计结果，没有误路由到送检量、质控或结果结构
- 还需继续优化的部分：
  - `topic` 当前仍可能显示为 `未识别问题`，没有完全回填为“PGT-A 整倍体率”
  - `route_trace` 里 `candidate_metric_ids` 还不够完整，调试信息可读性一般
  - 回答文案里会同时出现“本次查询结果”和“当前快照全年覆盖范围”，容易让老师误以为统计口径不是 `7月到10月`

### 10.4 当前建议验收口径

遇到这类问题时，人工验收优先看以下几点：

- 有没有正确识别到明确时间范围
- 有没有稳定落到 `pgta_euploid_rate`
- 有没有返回“这段时间总体整倍体率”的结果
- 有没有误答成“按月趋势”或其他指标

如果后续产品希望“整倍体率情况”默认输出按月变化，而不是这段时间总体值，需要单独补充路由策略，不应默认视为当前实现错误

## 11. 测试通过标准

满足以下条件即可认为第一版联调通过：

- 后端 health 和 metadata 正常
- 前端能正常打开并创建会话
- 5 个支持问题全部返回有效结果
- 模糊问题能进入 clarify，不默认执行
- 4 个不支持问题全部明确拒答
- 前端构建通过
