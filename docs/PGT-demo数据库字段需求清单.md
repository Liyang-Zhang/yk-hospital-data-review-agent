# PGT Demo 数据库字段需求清单

## 目的

本文档用于和 IT / 数据同事核对：为了支持 PGT 类产品 Demo 的自然语言查询、指标计算和报告生成，当前数据库至少需要能提供哪些信息。

本文档不假设数据库中的实际字段名，只描述需要提取的业务信息。

适用范围：

- 医院客户定期数据回顾 Demo
- 聚焦 `PGT-A`、`PGT-SR`、`PGT-M`
- 仅考虑查询和分析，不考虑写入

## 总体要求

为了让 Demo 能稳定回答问题，数据库侧至少要支持以下几类能力：

- 能唯一定位医院
- 能唯一定位周期
- 能唯一定位胚胎/样本
- 能判断产品类型
- 能识别时间范围
- 能取得检测结果分类
- 能取得分层分析所需属性
- 能识别特殊结果，如嵌合、意外发现
- 能标识记录是否有效、是否应纳入统计

## 一、基础筛选与关联信息

### 1. 医院信息

需要的信息：

- 当前记录所属医院
- 医院唯一标识
- 医院名称

用途：

- 按医院隔离数据
- 支持“查看我院数据”
- 作为所有查询的默认过滤条件

需要和 IT 核对：

- 医院信息是在周期表、报告表，还是客户表中
- 是否存在历史医院名称变更
- 一个记录是否可能关联多个医院

### 2. 周期信息

需要的信息：

- 周期唯一标识
- 周期所属医院
- 周期所属 PGT 产品类型
- 周期创建时间或送检时间
- 周期当前状态

用途：

- 统计送检周期数
- 作为季度、月度趋势分析基础
- 用于 PGT-SR 二阶段统计

需要和 IT 核对：

- 周期的业务定义是什么
- 是否存在撤销、重复创建、重测等情况
- 哪些状态应纳入正式统计

### 3. 胚胎 / 样本信息

需要的信息：

- 胚胎或样本唯一标识
- 所属周期
- 所属医院
- 所属产品类型
- 送检时间或检测时间
- 当前记录状态

用途：

- 统计送检胚胎数
- 计算整倍体率、扩增成功率等以胚胎为单位的指标

需要和 IT 核对：

- “样本”和“胚胎”是否是一一对应
- 一个胚胎是否可能存在多次检测记录
- 多次检测时统计口径取哪条记录

## 二、时间信息

### 4. 时间维度信息

需要的信息：

- 年
- 月
- 季度
- 报告时间或结果出具时间
- 送检时间
- 检测完成时间

用途：

- 支持按月、季度、年度统计
- 支持同比分析
- 支持趋势图

需要和 IT 核对：

- 业务汇报中通常按哪一个时间口径统计，是送检时间还是出报告时间
- 是否存在补录或回写导致时间失真的情况

## 三、产品分类信息

### 5. PGT 产品类型

需要的信息：

- 记录属于 `PGT-A`、`PGT-SR` 还是 `PGT-M`
- 是否有更细一级项目分类

用途：

- 统计产品分布
- 按产品线拆分趋势和指标

需要和 IT 核对：

- 产品类型是否有稳定枚举值
- 历史数据中是否存在命名不一致

## 四、通用检测过程信息

### 6. 扩增相关信息

需要的信息：

- 是否扩增成功
- 扩增失败原因
- 是否为 NA

用途：

- 计算扩增成功率
- 计算检测成功率
- 支持失败样本排查

需要和 IT 核对：

- 扩增成功和检测成功是否是两个独立状态
- NA 在库中的表现形式是什么

### 7. 检测成功信息

需要的信息：

- 是否检测成功
- 检测失败原因

用途：

- 计算检测成功率
- 与扩增成功率区分

需要和 IT 核对：

- 若扩增成功但检测失败，数据库如何记录
- 最终统计口径中失败样本是否保留

## 五、PGT-A 结果信息

### 8. PGT-A 结果分类

需要的信息：

- 是否整倍体
- 是否属于 `1-4` 处异常
- 是否属于 `≥5` 处异常
- 是否属于异倍体
- 是否属于 UPD

用途：

- 计算整倍体率
- 输出结果结构分布

需要和 IT 核对：

- 这些分类是数据库已有字段，还是需要从原始结果二次映射
- 同一胚胎是否只会落入一个分类

### 9. PGT-A 年龄分层分析信息

需要的信息：

- 母亲年龄，或能计算检测时年龄的出生日期

用途：

- 输出 `<35`、`35-37`、`38-40`、`41-43`、`≥44` 的整倍体率

需要和 IT 核对：

- 年龄是直接存储还是需要计算
- 年龄统计是按送检时点、取卵时点还是报告时点

### 10. PGT-A 嵌合信息

需要的信息：

- 是否嵌合
- 嵌合比例分组
- 是否属于 `30%-49%`
- 是否属于 `50%及以上`

用途：

- 统计嵌合数量
- 计算嵌合占总胚胎数比例

需要和 IT 核对：

- 嵌合信息是独立字段还是文本结论
- 若是文本，需要否可稳定结构化提取

### 11. PGT-A 意外发现信息

需要的信息：

- 是否存在意外发现
- 意外发现类型

用途：

- 统计意外发现数量
- 计算占整倍体数比例和占总胚胎数比例

需要和 IT 核对：

- 意外发现是否只出现在整倍体结果中
- 意外发现类型是否结构化存储

## 六、PGT-SR 结果信息

### 12. PGT-SR 整倍体结果

需要的信息：

- 是否整倍体

用途：

- 计算 PGT-SR 整倍体率

### 13. PGT-SR 二阶段信息

需要的信息：

- 是否可进入二阶段
- 是否实际进入二阶段
- 二阶段对应胚胎数
- 是否不携带

用途：

- 统计可进入二阶段周期数
- 统计实际进入二阶段周期数
- 统计不携带数与不携带率

需要和 IT 核对：

- “可进入二阶段”是按周期还是按胚胎判断
- “实际进入二阶段”是否存在人工补录

### 14. PGT-SR 异常类型信息

需要的信息：

- 平衡易位
- 罗氏易位
- 微缺失重复
- 臂间倒位
- 其他异常

用途：

- 输出异常类型分布
- 支持按类型拆分统计

需要和 IT 核对：

- 异常类型是标准枚举还是自由文本
- 一个周期是否可能同时命中多个异常类型

### 15. PGT-SR 特殊结果信息

需要的信息：

- 嵌合信息
- 意外发现信息

用途：

- 对齐人工汇报中的特殊结果统计

## 七、PGT-M 结果信息

### 16. PGT-M 整倍体结果结构

需要的信息：

- 是否整倍体
- 是否整倍体+不携带
- 是否整倍体+杂合
- 是否整倍体+纯合或双重杂合

用途：

- 计算 PGT-M 整倍体率
- 输出结果结构分布

需要和 IT 核对：

- 这些分类是否已有明确字段
- 是否能稳定映射为互斥类别

### 17. PGT-M 病种 / 基因信息

需要的信息：

- 致病基因名称
- 疾病名称
- 所属周期

用途：

- 统计病种构成
- 输出 TopN 病种 / 基因分布

需要和 IT 核对：

- 病种名称和基因名称是否标准化
- 是否存在同义词、历史写法不一致

### 18. PGT-M 特殊结果信息

需要的信息：

- 嵌合信息
- 意外发现信息

用途：

- 输出特殊发现统计

## 八、统计口径控制信息

### 19. 有效性控制信息

需要的信息：

- 记录是否有效
- 是否已取消
- 是否重测
- 是否重复报告
- 是否作废

用途：

- 过滤不应纳入统计的数据
- 避免重复计数

需要和 IT 核对：

- 年度汇报默认排除了哪些状态
- 同一个周期或胚胎存在多条记录时如何取数

### 20. 结果版本信息

需要的信息：

- 是否存在多版结果
- 最终生效版本标记

用途：

- 保证按最终结果统计

需要和 IT 核对：

- 历史修订记录如何保存
- 是否能直接识别最终版

## 九、Demo 第一版最少字段能力

如果 IT 需要先评估最小可行范围，第一版 Demo 最少需要能提供以下信息：

- 医院归属
- 周期唯一标识
- 胚胎或样本唯一标识
- 产品类型
- 时间信息
- 扩增成功状态
- 检测成功状态
- PGT-A 整倍体及异常分类
- 年龄信息
- PGT-SR 二阶段相关状态
- PGT-SR 异常类型
- PGT-M 病种或基因名称
- 嵌合信息
- 意外发现信息
- 有效/取消/重测状态

## 十、建议和 IT 核对的表级问题

可以直接让 IT 协助回答下面这些问题：

1. 哪些表能提供医院、周期、胚胎、产品类型、时间这几类核心信息？
2. 哪些表能提供 PGT-A / SR / M 的最终检测结果？
3. 特殊结果如嵌合、意外发现是在结构化字段中，还是在报告文本中？
4. 是否有单独的结果明细表可区分整倍体、异常类型、不携带、杂合、纯合等分类？
5. 一条记录的“最终有效结果”如何识别？
6. 是否已有面向业务统计的汇总视图，还是需要从原始明细表自行汇总？
7. 当前数据库中是否能直接按医院过滤？
8. 数据量级、查询耗时、建议的访问方式分别是什么？

## 十一、当前 Demo 直接依赖的指标

第一版建议只承诺以下指标，便于和 IT 优先对齐：

- PGT 总周期数、总胚胎数、同比
- PGT-A / SR / M 产品分布
- 月度、季度趋势
- 扩增成功率、检测成功率
- PGT-A 整倍体率
- PGT-A 年龄分层整倍体率
- PGT-SR 二阶段不携带率
- PGT-M 病种分布
- 嵌合占比
- 意外发现占比

如果上述指标所需信息不能结构化取得，则 Demo 范围需要同步缩小。
